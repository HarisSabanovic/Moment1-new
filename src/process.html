Projektstart

För att kicka igång mitt projekt börjar jag med att skapa en katalog. Inom denna katalog öppnar jag en terminal och initierar ett nytt npm-projekt genom att använda kommandot "npm init" och följer startsekvensen. npm är en pakethanterare som hanterar tillägg till nodejs.

Nästa steg är att lägga till Parcel-tillägget med kommandot "npm install parcel --save-dev". Detta görs eftersom Parcel enbart används under utvecklingsfasen och inte vid själva publiceringen. Dessutom indikerar det att Parcel används för projektets utveckling. Parcel sköter minifiering av filer vid publicering, samt transpilering av JavaScript för äldre webbläsarversioner. Det innebär att JavaScript-filerna görs körbara för äldre webbläsare som inte stödjer nyare JavaScript-funktioner. Även bildoptimering hanteras, förutsatt att det är specificerat i html-filerna.

I package.json specificerar jag källfilen för projektet, exempelvis index.html. Därefter skapar jag själva källfilen och en mapp kallad "src" för ytterligare moduler. När konfigurationen av npm-projektet är klar, öppnar jag Visual Studio Code (VSCode) och initierar Git för att påbörja versionshantering. Redan här byter jag till en gren som heter "dev" för att separera arbetsfiler från publiceringsfiler.

Projektutveckling

För att starta min utvecklingsmiljö öppnar jag en terminal och anger kommandot "npm start". Detta startar en lokal liveserver där jag kan se mina ändringar i realtid och öppnar det i ett webbläsarfönster. Därefter öppnar jag VSCode där den faktiska programmeringen sker.

Projektpublicering

För att publicera mitt projekt använder jag Netlify. Inom Netlify väljer jag vilket GitHub-repo som ska publiceras och går igenom inställningarna som krävs för att Netlify ska kunna bygga min webbplats.
